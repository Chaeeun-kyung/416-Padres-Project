# SeaWulf ReCom Wiring Guide

This guide shows how ReCom scripts connect to Prepro-6 outputs, including input and output formats.

## 1) Required Input Files

For each state, generated by `generate_seawulf_inputs.py`:

- `results/seawulf_inputs/<STATE>/<STATE>_graph.json`
  - Fields:
    - `state`
    - `node_count`
    - `edge_count`
    - `adjacency` (map of `GEOID -> [neighbor_GEOID, ...]`)

- `results/seawulf_inputs/<STATE>/<STATE>_precinct_attributes.json`
  - Array of precinct property objects
  - Important fields used by ReCom scripts:
    - `GEOID`
    - `district_number`
    - `votes_dem`, `votes_rep`, `votes_total`
    - `CVAP_TOT24` (population proxy if present)
    - Optional constrained-mode field, e.g. `CVAP_HSP24`

## 2) Driver Script

- File: `scripts/recom_driver.py`
- Purpose:
  - Runs a single-chain simplified ReCom over the real precinct graph.
  - Optionally enforces a VRA-style opportunity district floor.

### Example (race-blind)

```powershell
python scripts/recom_driver.py --state AZ --steps 100 --input-root results/seawulf_inputs --output results/recom_AZ_plans.json
```

### Example (constrained)

```powershell
python scripts/recom_driver.py --state AZ --steps 100 --input-root results/seawulf_inputs --output results/recom_AZ_constrained.json --vra-group-field CVAP_HSP24 --vra-min-districts 2 --vra-threshold 0.50
```

## 3) Worker Script (parallel core use case)

- File: `scripts/recom_worker.py`
- Purpose:
  - Runs one worker chain using same real graph input.
  - Writes `recom_worker_<STATE>_<worker>.json` by default.

### Example

```powershell
python scripts/recom_worker.py 0 25 --state AZ --input-root results/seawulf_inputs
```

## 4) Merge Script (aggregate core outputs)

- File: `scripts/merge_parallel_results.py`
- Purpose:
  - Merges worker plan files into one ensemble output.

### Example

```powershell
python scripts/merge_parallel_results.py --state AZ --pattern results/recom_worker_AZ_*.json --output results/recom_parallel_merged_AZ.json
```

## 5) Output Format Overview

Per plan entry (driver/worker outputs):

- `state`
- `worker` (worker file only; `null` for driver single-chain)
- `step`
- `district_populations` (map district -> summed node population proxy)
- `winners` (district -> `D` or `R`)
- `split` (`{"R": x, "D": y}`)
- `assignment` (`GEOID -> district_number`)

Merged output contains:

- `state`
- `summary` (avg/min/max seat splits)
- `plans`
- `source_files`

## 6) SeaWulf Batch Scripts

Updated scripts:

- `scripts/recom_driver.slurm`
- `scripts/recom_parallel.slurm`

Both scripts now support `STATE`, `INPUT_ROOT`, and optional constrained mode:

- `VRA_GROUP_FIELD`
- `VRA_MIN_DISTRICTS`
- `VRA_THRESHOLD`

This makes the same pipeline reusable for race-blind and constrained runs.
